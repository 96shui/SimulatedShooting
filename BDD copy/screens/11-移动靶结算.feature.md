# 11 移动靶结算

来源：draw.io 页面 `p11 11 移动靶结算`

## 功能

作为玩家，我希望移动目标训练结束后看到命中数量、弹药消耗、点射记录、路线命中时间点和评级。

```gherkin
# language: zh-CN
功能: 移动目标射击训练结算

  背景:
    假如移动目标射击训练已经结束
    那么系统应显示“移动目标射击 训练结算”

  场景大纲: 根据命中发数显示评级
    假如本局命中发数为 <命中数>
    当系统计算移动靶评级
    那么界面应显示评级“<评级>”

    例子:
      | 命中数 | 评级 |
      | 0 | 不及格 |
      | 2 | 不及格 |
      | 3 | 及格 |
      | 4 | 良好 |
      | 5 | 优秀 |
      | 8 | 优秀 |

  场景: 显示训练摘要
    当结算界面加载完成
    那么界面应显示命中数
    而且应显示消耗弹药 10 发
    而且应显示昼夜模式和目标速度
    而且应显示评级规则

  场景: 显示 5 次点射记录
    当结算界面加载完成
    那么界面应显示第 1 次到第 5 次点射记录
    而且每次记录应标明命中或未命中
    而且路线命中时间点应与点射记录一致

  场景: 重新训练
    当玩家点击“重新训练”
    那么系统应清空本次移动靶 Session
    而且返回移动靶设置界面或使用上次设置直接重新开始

  场景: 返回模式选择
    当玩家点击“返回模式选择”
    那么系统应返回训练模式选择界面
    而且玩家档案应保存本次移动靶评级摘要
```

## Unity 验收点

- 结算页所有数值来自 `MovingTargetSession`，不能使用静态示例文本。
- 路线命中时间点建议保存为可序列化记录，便于回放和测试。
