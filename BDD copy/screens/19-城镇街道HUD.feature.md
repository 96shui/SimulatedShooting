# 19 城镇街道HUD

来源：draw.io 页面 `p19 19 城镇街道HUD`

## 功能

作为玩家，我希望在城镇街道 HUD 中查看小地图、建筑入口、任务提示、弹药和姿态状态，并进入可搜索建筑。

```gherkin
# language: zh-CN
功能: 城镇街道 HUD

  背景:
    假如玩家已进入城镇攻防街道场景
    而且玩家与两名队友保持纵队
    那么系统应显示城镇街道第一人称 HUD

  场景: HUD 显示街道任务信息
    当城镇街道 HUD 加载完成
    那么小地图应显示玩家、队友和建筑入口
    而且小地图应显示红色敌情预估
    而且任务提示应显示进入建筑、搜索房间、消灭敌人
    而且弹药应显示 30 / 120

  场景: 建筑入口显示可交互提示
    假如玩家靠近可进入建筑入口
    当玩家视线或 XR 射线指向入口
    那么界面应显示“建筑入口 / 可交互提示”
    而且入口高亮不应遮挡战斗 HUD

  场景: 进入建筑搜索阶段
    假如玩家位于建筑入口交互范围内
    当玩家确认进入
    那么系统应切换到建筑内部场景或楼层区域
    而且 HUD 应切换为“建筑搜索HUD”
    而且街道已清除状态应保存在城镇 Session 中

  场景: 街道敌人未清除时提示风险
    假如街道敌人仍存活
    当玩家尝试进入建筑
    那么系统应允许进入或提示风险，取决于项目规则
    而且任务状态应保留未清除的街道敌人

  场景: 姿态、换肩和拐角状态显示
    当玩家切换姿态或换肩状态
    那么底部状态栏应实时更新姿态、右肩/左肩和拐角射击可用状态
```

## Unity 验收点

- 建筑入口建议使用触发器和可交互组件双重判断：进入范围后才允许确认。
- 街道敌人清除状态、建筑入口状态和玩家位置应属于 `UrbanSession`。
