# 01 通用视觉风格

来源：draw.io 页面 `p01 01 通用视觉风格`

## 功能

作为玩家，我希望所有训练界面使用统一的军事 VR HUD 风格，以便在不同训练模式之间获得一致、清晰、可读的操作体验。

```gherkin
# language: zh-CN
功能: 通用视觉风格

  背景:
    假如玩家进入任意 VR 训练 UI 页面
    而且系统使用深灰黑背景、青蓝 HUD 线框和少量橙色警示色

  场景: 页面加载时显示统一的边框布局
    当页面初始化完成
    那么屏幕应显示主 UI 边框
    而且素材区域应作为可替换的占位区域显示
    而且非交互文本应显示在带边框的文本容器内
    但是素材占位不应被当作可点击按钮

  场景: 左下状态面板展示玩家档案摘要
    假如玩家已有训练档案
    当通用 HUD 状态面板刷新
    那么左下状态面板应显示玩家档案、训练等级和最近评级
    而且显示内容应来自玩家档案数据

  场景: 右侧信息栏展示当前页面提示
    假如当前页面存在任务入口、系统提示或状态数据
    当页面刷新右侧信息栏
    那么右侧信息栏应按照分组显示这些信息
    而且警示内容应使用橙色或红色强调

  场景: VR 视野中 UI 不遮挡核心内容
    假如玩家在头显中查看 UI
    当玩家轻微转动头部
    那么主操作按钮和关键文本仍应在舒适视野范围内
    而且 HUD 不应遮挡射击视线中心区域
```

## Unity 验收点

- 页面根节点建议使用 `Canvas` 或 World Space Canvas，所有面板有稳定对象名。
- 非交互文本使用 `RectTransform + Image + TMP_Text`，按钮使用独立 Button 或 XR UI 交互组件。
- 素材占位后续可替换为 RenderTexture、3D 模型或场景相机，但测试 ID 保持不变。
