# 14 堑壕开场简报

来源：draw.io 页面 `p14 14 堑壕开场简报`

## 功能

作为玩家，我希望在进入堑壕前看到无人机投射的堑壕平面图、小队配置和任务目标。

```gherkin
# language: zh-CN
功能: 堑壕射击开场任务

  背景:
    假如玩家已选择堑壕地图和武器
    当堑壕任务准备完成
    那么系统应显示“堑壕射击 开场任务”

  场景: 播放无人机起飞并显示地图投影
    当开场简报开始
    那么系统应播放无人机起飞或等效过场
    而且投影区域应显示堑壕入口、队友位置和红色敌情区域
    而且玩家在简报期间不应提前进入堑壕战斗状态

  场景: 显示小队配置
    当简报界面加载完成
    那么左侧应显示玩家在最前
    而且显示二号队友后方掩护
    而且显示三号队友朝后警戒

  场景: 显示任务目标
    当玩家查看任务目标面板
    那么界面应显示进入堑壕
    而且显示搜索路线
    而且显示消灭敌人
    而且显示保持小队队形

  场景: 开始进入堑壕
    当玩家选择“开始进入”
    那么系统应创建堑壕任务 Session
    而且生成 3 到 5 名敌人
    而且生成两名队友并排列为三人纵队
    而且进入“堑壕第一人称HUD”

  场景: 查看地图
    当玩家选择“查看地图”
    那么系统应放大或聚焦堑壕平面图
    而且玩家应能返回开场简报

  场景: 返回武器库
    当玩家选择“返回武器库”
    那么系统应返回武器库界面
    而且不应创建新的堑壕任务 Session
```

## Unity 验收点

- 开始进入时才创建敌人和队友，简报阶段只显示配置预览。
- 队友队形可通过 NavMeshAgent、路径点或自定义跟随逻辑实现，但状态需可测试。
