# 20 建筑搜索HUD

来源：draw.io 页面 `p20 20 建筑搜索HUD`

## 功能

作为玩家，我希望在建筑内部搜索房间时查看楼层小地图、房间状态、危险区域、交互提示和任务进度。

```gherkin
# language: zh-CN
功能: 城镇建筑搜索 HUD

  背景:
    假如玩家已经进入城镇建筑内部
    那么系统应显示建筑搜索 HUD

  场景: HUD 显示楼层搜索信息
    当建筑搜索 HUD 加载完成
    那么左侧楼层小地图应显示已搜索房间和未搜索房间
    而且显示队友位置
    而且显示敌人可能区域
    而且右侧任务面板应显示搜索房间、清除敌情、队友跟随

  场景: 房门可交互时显示提示
    假如玩家靠近未搜索房间门口
    当玩家视线或 XR 射线指向房门
    那么中间交互提示应显示开门、观察、拐角伸枪射击
    而且玩家应能选择其中一个交互动作

  场景: 开门后更新房间状态
    假如玩家位于未搜索房间门口
    当玩家执行“开门”
    那么房门应打开
    而且房间状态应从未搜索变为搜索中
    而且若房间内生成敌人，敌人应进入可遭遇状态

  场景: 观察房间不暴露玩家身体
    假如玩家位于门口或拐角观察点
    当玩家执行“观察”
    那么系统应允许玩家查看房间或拐角后方
    而且玩家身体碰撞体不应完全进入危险区域

  场景: 拐角伸枪射击
    假如 HUD 显示危险区域提示
    当玩家执行“拐角伸枪射击”
    那么武器应允许从拐角外侧发射
    而且玩家主体位置应保持在掩体后
    而且命中敌人时应播放中弹音效

  场景: 房间清空后标记已搜索
    假如房间内敌人已全部消灭
    当玩家完成房间检查
    那么楼层小地图应将该房间标记为已搜索
    而且城镇 Session 的搜索进度应增加

  场景: 搜索完成后进入结算
    假如所有可搜索房间已搜索
    而且所有敌人已消灭
    当任务状态刷新
    那么系统应进入“城镇攻防 任务结算”
    而且任务结果应为胜利
```

## Unity 验收点

- 房间、房门、危险区域和搜索状态建议使用明确数据模型，避免只依赖场景物体激活状态。
- 拐角伸枪射击要校验武器射线和玩家碰撞体，防止穿墙命中。
