# 18 城镇地图选择

来源：draw.io 页面 `p18 18 城镇地图选择`

## 功能

作为玩家，我希望在城镇攻防前选择地图，并了解街道敌人、建筑敌人、房间搜索和失败条件。

```gherkin
# language: zh-CN
功能: 城镇攻防地图选择

  背景:
    假如玩家选择了城镇攻防
    那么系统应显示“城镇攻防 地图选择”

  场景: 默认选中城镇地图 A
    当地图选择界面加载完成
    那么“城镇地图 A”应处于选中状态
    而且显示街道敌人 1-2
    而且显示建筑敌人 3-6

  场景大纲: 切换城镇地图
    当玩家选择“<地图>”
    那么“<地图>”应显示选中边框
    而且另一个地图卡片应取消选中
    而且当前待确认地图应为“<地图>”

    例子:
      | 地图 |
      | 城镇地图 A |
      | 城镇地图 B |

  场景: 显示城镇任务信息
    当玩家查看任务信息面板
    那么界面应显示搜索楼房
    而且显示消灭所有敌人
    而且显示队友保持纵队
    而且显示失败条件为玩家死亡

  场景: 选择地图进入城镇任务
    假如玩家已选择城镇地图 A
    当玩家点击“选择地图”
    那么系统应保存当前城镇地图选择
    而且创建或准备城镇攻防 Session
    而且后续进入街道场景时应生成 1 到 2 名街道敌人
    而且建筑内应生成 3 到 6 名敌人

  场景: 返回上一级
    当玩家点击“返回”
    那么系统应返回训练模式选择或主菜单
    而且不应创建城镇任务 Session
```

## Unity 验收点

- 城镇地图配置应包含街道入口、建筑入口、楼层房间、敌情区域和搜索节点。
- 地图选择只保存配置，敌人和房间状态在 Session 中初始化。
