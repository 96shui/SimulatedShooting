# 06 弹着分析

来源：draw.io 页面 `p06 06 弹着分析`

## 功能

作为玩家，我希望每轮射击后看到垂直和水平偏差，并根据提示应用准星柱和觇孔调整。

```gherkin
# language: zh-CN
功能: 本轮弹着分析

  背景:
    假如玩家完成 100m 射校的一轮 3 发射击
    那么系统应打开本轮弹着分析弹窗

  场景: 显示本轮弹着点和偏差数据
    当弹着分析界面加载完成
    那么左侧应显示 50cm×50cm 胸靶示意
    而且胸靶上应显示本轮 3 个弹着点
    而且右侧应显示垂直偏差和水平偏差
    而且右侧应显示准星柱调整方向和角度
    而且右侧应显示觇孔调整方向和格数

  场景大纲: 根据垂直偏差给出准星柱调整方向
    假如本轮弹着中心相对靶心“<偏差方向>”
    当系统计算准星柱建议
    那么界面应提示“<调整建议>”

    例子:
      | 偏差方向 | 调整建议 |
      | 偏上 | 准星柱逆时针调整 X° |
      | 偏下 | 准星柱顺时针调整 X° |

  场景大纲: 根据水平偏差给出觇孔调整方向
    假如本轮弹着中心相对靶心“<偏差方向>”
    当系统计算觇孔建议
    那么界面应提示“<调整建议>”

    例子:
      | 偏差方向 | 调整建议 |
      | 偏左 | 觇孔向前调整 X 格 |
      | 偏右 | 觇孔向后调整 X 格 |

  场景: 应用调整后更新归零状态
    当玩家点击“应用调整”
    那么系统应把本轮建议应用到武器归零状态
    而且界面应标记调整已应用
    而且重复点击不应重复叠加同一次调整

  场景: 进入下一轮
    假如本轮调整已应用
    当玩家点击“进入下一轮”
    那么系统应回到“100m射击HUD”
    而且轮次应增加 1
    而且本轮弹数应重置为 3/3

  场景: 第三轮后进入最终评级
    假如当前已经是第 3 轮
    当玩家完成分析并继续
    那么系统应进入“射校最终评级”界面
```

## Unity 验收点

- 偏差算法应独立成可单元测试服务：垂直 23cm 一周，0.064cm 约等于 1°；水平一格 2cm 并向上取整。
- `应用调整` 需要幂等，避免玩家多次点击造成调整量重复累加。
