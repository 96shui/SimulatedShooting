# 任务规格与分工

本目录用于按产品阶段和四人分工管理开发任务。每个任务必须是可独立实现、可独立验收的工作单元；需要联调的地方必须在任务规格中明确联调对象和联调接口。

## 阶段目录

| 阶段 | 目录 | 范围 |
|---|---|---|
| P1 | [阶段一-P1-100m射校原型闭环](阶段一-P1-100m射校原型闭环) | 100m 射校闭环、基础输入、基础武器、HUD、弹着分析、结算 |
| P2 | [阶段二-P2-移动靶与全局系统扩展](阶段二-P2-移动靶与全局系统扩展) | 移动靶、武器库原型、设置原型、弹药/HUD 复用 |
| P3 | [阶段三-P3-小型堑壕与城镇战斗原型](阶段三-P3-小型堑壕与城镇战斗原型) | 小型堑壕、小型城镇、简单敌人 AI、3 人纵队、简单血量 |

## 固定分工目录

每个阶段都包含四个负责人目录：

- `UI`
- `场景`
- `功能A-玩法流程与规则`
- `功能B-武器输入与AI基础`

## 文件命名

- 每个负责人目录必须有 `任务清单.md`。
- 具体任务规格文件使用 `task001-任务简称.md` 格式。
- 任务编号在同一阶段内全局唯一，避免不同负责人目录出现重复编号。
- 任务规格必须写明 BDD 引用、接口文档引用、交付物、测试要求和联调对象。
