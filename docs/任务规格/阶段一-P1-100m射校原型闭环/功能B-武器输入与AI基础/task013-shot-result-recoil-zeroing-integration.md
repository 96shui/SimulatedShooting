# task013 射击结果、后坐力与弹着数据接入

## 负责人

功能B-武器输入与AI基础

## 目标

把基础武器的开火、后坐力、稳定度、命中点和弹药变化接入 100m 射校服务，形成从输入到弹着记录的完整链路。

- `docs/BDD/screens/05-100m射击HUD.feature.md`
- `docs/接口文档/04-100m射校服务.md`
- `docs/接口文档/06-武器与弹药服务.md`
- `docs/接口文档/03-HUD显示数据.md`

## 交付内容

- 射击结果对象或事件：
  - SessionId。
  - 命中点或瞄准点。
  - 稳定度。
  - 当前肩侧。
  - 弹药消耗结果。
- 简单后坐力：
  - 可被玩家感知。
  - 不破坏 P1 测试可重复性。
- 稳定度计算原型：
  - 可基于武器/控制器晃动或测试输入值。
  - PlayMode Test 可注入稳定度。
- 调用 `IZeroingService.RegisterShot` 或等效命令。

## 不包含

- 不做真实枪械后坐力曲线。
- 不做高级弹道学。

## 依赖关系

- 前置依赖：task005、task006、task012。
- 后续依赖：task014、task015、task016。

## 联调说明

- 与 功能A 联调：射击结果字段必须满足射校服务入参。
- 与 场景 联调：靶标命中坐标转换。
- 与 UI 联调：弹药、稳定度、弹着记录刷新。

## 测试要求

- EditMode 单元测试：
  - 稳定度输入影响散布/偏移输入的规则。
  - 禁射或无弹时不产生射击结果。
- PlayMode 测试：
  - 输入替身开火后 HUD 弹数减少。
  - 命中靶标后射校服务记录弹着。
  - 切肩后射击结果肩侧正确。

## 验收标准

- 一次开火能完整触发弹药消耗、命中检测、射校记录和 HUD 刷新。
- 后坐力可见但不造成自动化测试不稳定。
- 无 VR 输入替身能完成 3 发一轮。
- 相关 EditMode/PlayMode 测试通过。
