# task014 P1 流程 PlayMode 测试与联调门禁

## 负责人

功能A-玩法流程与规则

## 目标

建立 P1 端到端 PlayMode 测试和联调门禁，验证四个人的工作能串成完整 100m 射校闭环。

## 交付内容

- P1 PlayMode 测试场景或测试夹具。
- 输入替身驱动完整流程：
  - 主菜单进入 100m。
  - 开始训练。
  - 完成 3 发射击。
  - 打开弹着分析。
  - 应用调整。
  - 进入下一轮或结算。
- 至少一个可控结果测试：
  - 第 1 轮通过输出优秀。
  - 3 轮未通过输出不及格。
- 测试失败时能定位到 UI、场景、功能A或功能B责任边界。

## 不包含

- 不替代各任务自己的单元测试。
- 不做 VR 实机手工验收记录。

## 依赖关系

- 前置依赖：task002-task013。
- 后续依赖：task015、task016。

## 联调说明

- 与 UI 联调：测试 ID 和页面状态。
- 与 场景 联调：场景加载、靶标可查找。
- 与 功能B 联调：输入替身、射击结果、弹药状态。

## 测试要求

- PlayMode 测试必须能在无 VR 设备环境运行。
- 测试不依赖真实随机结果，使用固定随机种子或测试数据。
- 每个测试名或说明引用对应 BDD 场景。

## 验收标准

- P1 核心 PlayMode 流程测试通过。
- 失败用例能明确提示缺失的页面、服务、场景对象或输入事件。
- 该任务完成后，P1 其余任务才能进入最终收口。
