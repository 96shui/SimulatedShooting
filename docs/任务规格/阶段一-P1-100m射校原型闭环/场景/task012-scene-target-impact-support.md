# task012 靶标命中与弹着可视化场景支持

## 负责人

场景

## 目标

为 P1 100m 射校提供可命中的靶标、弹着点可视化和场景标识，使功能B的射线/弹道和功能A的弹着记录可以联动。

## 参考资料

- `UI/Sample/vr-shooting-100m-zeroing-first-person-hud-ui.png`
- `UI/Sample/vr-shooting-100m-impact-analysis-ui.png`
- `docs/BDD/screens/05-100m射击HUD.feature.md`
- `docs/BDD/screens/06-弹着分析.feature.md`

## 交付内容

- 胸靶碰撞体或射线检测面。
- 靶心坐标和 10 环半径配置。
- 弹着点可视化：
  - 场景靶面命中点。
  - 可传给 UI 缩略图/分析图的数据来源。
- 稳定命名：
  - 靶标根节点。
  - 靶面。
  - 靶心参考点。
- 命中区域调试 Gizmo 或调试显示。

## 不包含

- 不计算偏差调整规则。
- 不实现完整子弹物理。

## 依赖关系

- 前置依赖：task004、task005、task006。
- 后续依赖：task013、task014、task016。

## 联调说明

- 与 功能B 联调：射线/弹道命中靶面后返回命中点。
- 与 功能A 联调：靶面坐标转换为厘米单位偏差。
- 与 UI 联调：弹着点在 HUD/分析图中一致。

## 测试要求

- PlayMode 测试：
  - 射线命中靶面返回稳定坐标。
  - 靶心命中偏差接近 0。
  - 超出靶面命中被正确拒绝或标记。

## 验收标准

- 靶标命中坐标可稳定转换为射校服务输入。
- 场景显示弹着点与分析 UI 数据一致。
- 靶标对象和参考点可被测试查找。
- PlayMode 测试通过。
