# task016 P1 场景视觉、性能与联调收口

## 负责人

场景

## 目标

对 P1 100m 靶场进行视觉、性能和联调收口，保证场景风格与 UI 参考图统一，并能稳定支撑无 VR 与 VR 测试。

## 参考资料

- `UI/Sample/vr-shooting-100m-zeroing-first-person-hud-ui.png`
- `UI/Sample/vr-shooting-100m-zeroing-briefing-ui.png`
- `docs/需求/阶段化整体需求说明书.md`
- `docs/接口文档/11-Unity场景与Prefab约定.md`

## 交付内容

- P1 靶场视觉整理：
  - 基础材质。
  - 合理光照。
  - 射击位和靶标识别。
  - 参考图风格对齐。
- 性能基础检查：
  - 控制模型面数和贴图尺寸。
  - 避免过重实时光和后处理。
  - 保持 VR 原型流畅。
- 场景联调检查：
  - UI/HUD 不被遮挡。
  - 武器射击点和靶标命中正常。
  - 测试相机/VR Origin 正常。
- 输出 P1 场景验收说明。

## 不包含

- 不做最终美术资产。
- 不做移动靶、堑壕、城镇。

## 依赖关系

- 前置依赖：task004、task012、task014、task015。
- 后续依赖：P1 总体验收。

## 联调说明

- 与 UI 联调：HUD 空间位置、字体可读、参考图一致性。
- 与 功能A 联调：场景加载和测试 ID。
- 与 功能B 联调：射击、后坐力、靶标命中。

## 测试要求

- PlayMode 测试：
  - 场景加载无错误。
  - 靶标、射击位、VR Origin、关键 UI 可查找。
  - 一次完整 P1 流程能在该场景内运行。
- 手工验收：
  - 无 VR Editor 视角可操作。
  - VR 设备到位后检查 HUD 可读和射击位舒适度。

## 验收标准

- P1 靶场视觉符合写实训练原型方向，并和 UI 参考图风格不冲突。
- 场景对象命名和测试 ID 满足自动化测试。
- 没有明显性能风险或视线遮挡。
- P1 PlayMode 流程测试通过。
