## 权威规则

- 产品范围和阶段优先级以 `需求/阶段化整体需求说明书.md` 为准。
- 原始玩法规则以 `需求/VR射击游戏需求文档_汇总版.md` 为基础。
- 行为验收以 `BDD/` 为准。
- 前后端数据交互、DTO、事件、服务方法以 `接口文档/` 为准。
- UI 页面结构以 `../UI/Sample/vr-shooting-ui-reference-wireframes.drawio` 为准。

## BDD 工作流

任何功能实现前，先确认或更新对应 BDD，再确认或更新接口文档，最后实现代码、场景和测试。UI 不直接计算评级、胜负、敌人生成、弹药规则或训练结算；这些行为必须来自服务层 DTO、命令或事件。

## 功能交付门禁

每个拆分出来的功能都必须带测试交付，不能只交付实现代码：

- 业务规则、DTO、算法、状态机、服务命令和错误码必须有 EditMode 单元测试。
- UI、场景、HUD、页面跳转、XR 输入替身和训练闭环必须有 PlayMode 测试。
- 所有测试必须能追溯到 `BDD/screens/*.feature.md` 中的一个或多个场景。
- VR 设备相关功能必须同时保留无 VR 设备可运行的测试输入路径。
- 测试缺失时，该功能不得标记为完成。
