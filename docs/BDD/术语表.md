# BDD 术语表

本文档定义 BDD 场景中使用的统一语言，避免策划、开发和 QA 对同一行为使用不同词汇。

## 角色

| 术语 | 含义 |
|---|---|
| 玩家 | 佩戴 VR 头显并使用手柄或手部追踪进行训练的用户 |
| 队友 | 堑壕和城镇模式中跟随玩家的 AI 队友 |
| 二号队友 | 玩家后方第一名队友，换弹时可上前掩护 |
| 三号队友 | 队列最后一名队友，默认朝后警戒 |
| 敌人 | 训练任务中的 AI 对手，可被命中、击倒，尸体不消失 |

## UI 与交互

| 术语 | 含义 |
|---|---|
| 选择 | 玩家用 XR 射线或近场交互将焦点落到 UI 项上 |
| 确认 | 玩家按下确认输入，触发按钮或卡片的主行为 |
| 返回 | 玩家触发返回按钮或系统返回输入，回到上一层界面 |
| HUD | 战斗或训练中的头显叠加信息，包括弹药、任务、地图、状态 |
| 素材占位 | draw.io 中保留给 3D 场景、模型、地图或图标的区域，不作为 UI 按钮 |
| 文本框 | 非交互文本显示容器，对应 Unity 中一个 RectTransform 背板加 TMP_Text |
| 按钮 | 可交互 UI 控件，对应 Unity Button、XR Simple Interactable 或自定义 XR UI 控件 |

## 训练模式

| 术语 | 含义 |
|---|---|
| 100m精度射校靶 | 单发射击训练，3 发一轮，最多 3 轮，根据弹着偏差调整准星柱和觇孔 |
| 移动目标射击 | 侧身跑步靶移动训练，10 发弹，5 次点射，支持白天和夜晚模式 |
| 堑壕射击 | 玩家与两名队友进入堑壕，搜索并消灭 3 到 5 名敌人 |
| 城镇攻防 | 玩家与队友进入街道和三层楼房，搜索房间并消灭敌人 |
| 武器库 | 训练前选择武器的界面，支持 4 种武器 |
| 设置 | 调整 VR 舒适度、移动方式、转向方式、亮度、HUD 透明度、音量和语言 |

## 关键状态

| 术语 | 含义 |
|---|---|
| 可射击 | 当前规则允许扳机输入生成有效射击 |
| 禁止射击 | 当前规则忽略扳机输入，不消耗弹药，不记录命中 |
| 据枪稳定 | 枪体晃动低于阈值，允许高质量射击或显示稳定提示 |
| 本轮弹着分析 | 100m 射校每轮 3 发后出现的分析弹窗 |
| 应用调整 | 将本轮计算出的准星柱和觇孔建议应用到武器归零状态 |
| 任务结算 | 训练完成或失败后展示结果数据的界面 |
