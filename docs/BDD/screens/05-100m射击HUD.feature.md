# 05 100m射击HUD

来源：draw.io 页面 `p05 05 100m射击HUD`

阶段：P1。

## 功能

作为玩家，我希望在第一人称射击 HUD 中完成每轮 3 发射击，并实时看到轮次、距离、弹数、稳定度和弹着记录。

```gherkin
# language: zh-CN
功能: 100m 射击 HUD

  背景:
    假如玩家已开始 100m 精度射校靶
    而且当前轮次为 1/3
    而且本轮剩余弹数为 3/3

  场景: HUD 显示当前训练状态
    当 100m 射击 HUD 加载完成
    那么顶部左侧应显示“轮次 1/3”
    而且顶部中间应显示“距离 100m”
    而且顶部右侧应显示“弹数 3/3”
    而且右侧应显示弹着记录面板
    而且左下应显示稳定度条

  场景: 据枪稳定提示随枪体晃动变化
    假如玩家手持武器晃动低于稳定阈值
    当 HUD 刷新稳定度
    那么稳定度条应显示稳定状态
    而且中心提示应显示“稳定据枪”

  场景: 玩家单发射击后减少弹数
    当玩家扣动扳机完成一次有效射击
    那么本轮剩余弹数应减少 1
    而且弹着记录面板应追加一个命中点
    而且弹着点应由准星、枪体晃动和本局偏移共同计算

  场景: 无 VR 设备时用输入替身完成单发射击
    假如当前使用输入替身或 XR Device Simulator
    当测试输入触发扳机
    那么系统应按同一武器发射流程完成一次有效射击
    而且本轮剩余弹数应减少 1
    而且 HUD 刷新来源应仍然是服务层 DTO 或事件

  场景: 按按钮换弹和切换左右肩
    当玩家触发换弹输入
    那么武器应进入按钮换弹流程
    而且 HUD 弹药数据应从弹药服务刷新
    当玩家触发左右肩切换输入
    那么武器射击参考点应切换到另一侧
    而且 HUD 或操作提示应显示当前肩侧

  场景: 本轮 3 发完成后打开弹着分析
    假如玩家已完成本轮第 3 发有效射击
    当弹着记录写入完成
    那么系统应打开“本轮弹着分析”界面
    而且射击输入应临时禁用

  场景: 弹数为 0 时禁止继续射击
    假如本轮剩余弹数为 0
    当玩家继续扣动扳机
    那么系统不应生成新子弹
    而且弹着记录不应新增命中点
```

## Unity 验收点

- P1 允许使用基础弹道或射线命中接入训练 Session；后续可替换为更完整的 `Firearm`/`Bullet` 实现。
- 弹着记录不应只由子弹碰撞决定，还应记录射击时刻的准星状态和当前归零偏移。
- 同一套射击、换弹、左右肩切换逻辑必须同时支持真实 XR 输入和无 VR 输入替身。
