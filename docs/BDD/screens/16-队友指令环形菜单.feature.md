# 16 队友指令环形菜单

来源：draw.io 页面 `p16 16 队友指令环形菜单`

阶段：Later；P3 仅要求 3 人纵队跟随，不要求完成队友指令环形菜单。

## 功能

作为玩家，我希望在堑壕或城镇战斗中打开队友指令环形菜单，快速指挥队友投掷手榴弹、火力支援、掩护换弹、前进或停止。

```gherkin
# language: zh-CN
功能: 队友指令环形菜单

  背景:
    假如玩家处于支持队友指令的战斗场景
    而且至少一名队友处于可接收指令状态

  场景: 打开队友指令菜单
    当玩家按住队友指令输入
    那么屏幕中央应显示“队友指令”环形菜单
    而且菜单应包含手榴弹、火力支援、掩护换弹、前进、停止
    而且原有小地图、弹药和队友状态仍应可见

  场景大纲: 选择队友指令
    假如队友指令菜单已打开
    当玩家选择“<指令>”
    那么“<指令>”应显示选中高亮
    当玩家确认选择
    那么系统应向队友 AI 下发“<指令>”指令
    而且队友状态应更新为对应执行状态

    例子:
      | 指令 |
      | 手榴弹 |
      | 火力支援 |
      | 掩护换弹 |
      | 前进 |
      | 停止 |

  场景: 手榴弹指令需要有效目标区域
    假如没有可投掷的敌情区域
    当玩家选择并确认“手榴弹”
    那么系统应提示当前无有效投掷目标
    而且队友不应消耗手榴弹

  场景: 火力支援指令使队友压制目标方向
    假如前方存在敌人预估区域
    当玩家确认“火力支援”
    那么队友应进入压制射击状态
    而且玩家 HUD 应显示队友状态为执行火力支援

  场景: 掩护换弹指令保护玩家换弹
    假如玩家正在换弹或准备换弹
    当玩家确认“掩护换弹”
    那么二号队友应移动到掩护点或进入掩护姿态
    而且玩家换弹期间队友不应阻挡玩家视线和枪口

  场景: 取消队友指令菜单
    当玩家松开指令输入且未确认任何选项
    那么队友指令菜单应关闭
    而且不应下发任何队友指令
```

## Unity 验收点

- 队友指令建议通过 `SquadCommandService` 或 AI Blackboard 下发，UI 不直接控制动画。
- 指令失败需要可见反馈，避免玩家误以为指令已执行。
- 本页面不进入 P1/P2/P3 的完成门禁，除非重新调整阶段范围。
