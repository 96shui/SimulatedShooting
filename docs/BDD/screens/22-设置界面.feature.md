# 22 设置界面

来源：draw.io 页面 `p22 22 设置界面`

阶段：P2 设置原型；P1 可只提供必要默认设置。

## 功能

作为玩家，我希望在设置界面调整 VR 舒适度、移动方式、转向方式、亮度、HUD 透明度、音量、瞄准辅助和语言，并能实时预览 HUD 效果。

```gherkin
# language: zh-CN
功能: 设置界面

  背景:
    假如玩家从主菜单进入设置
    那么系统应显示设置界面
    而且设置项应加载当前保存值

  场景: 显示当前设置
    当设置界面加载完成
    那么界面应显示 VR 舒适度
    而且显示转向方式
    而且显示移动方式
    而且显示画面亮度滑条
    而且显示 HUD 透明度滑条
    而且显示音效音量滑条
    而且显示瞄准辅助开关
    而且显示语言

  场景大纲: 修改二选一或多选设置
    当玩家将“<设置项>”改为“<设置值>”
    那么界面应立即显示“<设置值>”为当前临时设置
    而且 HUD 实时预览窗口应刷新相关效果

    例子:
      | 设置项 | 设置值 |
      | VR舒适度 | 中 |
      | 转向方式 | 平滑 |
      | 转向方式 | 分段 |
      | 移动方式 | 瞬移 |
      | 移动方式 | 连续 |
      | 瞄准辅助 | 开 |
      | 瞄准辅助 | 关 |
      | 语言 | 中文 |

  场景大纲: 修改滑条设置
    当玩家调整“<设置项>”滑条到 <数值>
    那么实时预览窗口应反映该数值
    而且当前临时设置应保存为 <数值>

    例子:
      | 设置项 | 数值 |
      | 画面亮度 | 70 |
      | HUD透明度 | 60 |
      | 音效音量 | 50 |

  场景: 应用设置
    假如玩家修改了一个或多个设置项
    当玩家点击“应用”
    那么系统应保存当前设置
    而且 VR 移动、转向、HUD、音量和语言应使用新设置
    而且再次进入设置界面时应显示保存后的值

  场景: 恢复默认设置
    当玩家点击“恢复默认”
    那么界面应将所有设置项恢复为默认值
    而且实时预览窗口应刷新为默认效果
    而且只有玩家点击“应用”后默认值才应写入持久化配置

  场景: 返回不保存临时修改
    假如玩家修改了设置但未点击“应用”
    当玩家点击“返回”
    那么系统应返回上一界面
    而且未应用的临时修改不应写入持久化配置

  场景: 设置界面在训练中打开
    假如玩家从训练暂停菜单进入设置
    当玩家点击“返回”
    那么系统应回到训练暂停菜单
    而且不应丢失当前训练 Session
```

## Unity 验收点

- 设置应由 `SettingsService` 持久化，UI 只维护临时副本。
- 移动方式和转向方式应接入 XR Interaction Toolkit 的移动/转向 Provider。
- HUD 透明度和亮度变化应能被 PlayMode Test 观察到。
