# 02 游戏主界面

来源：draw.io 页面 `p02 02 游戏主界面`

阶段：P1 覆盖 100m 入口；P2/P3 再开放移动靶、堑壕、城镇等入口的完整流程。

## 功能

作为玩家，我希望从主界面进入不同训练模式、武器库和设置，并能看到自己的档案状态。

```gherkin
# language: zh-CN
功能: 游戏主界面

  背景:
    假如玩家启动游戏并完成基础加载
    那么系统应显示“VR射击训练系统 DEMO”主界面

  场景: 主界面显示玩家档案和底部状态栏
    当主界面加载完成
    那么左下角应显示玩家档案、训练等级和最近评级
    而且底部状态栏应显示网络、音量、用户、版本和提示
    而且这些信息不应阻挡右侧主菜单按钮

  场景: 进入 100m 精度射校靶
    当玩家选择“100m精度射校靶”
    那么系统应进入“100m任务说明”界面
    而且当前训练模式应记录为 100m 精度射校靶

  场景: 无 VR 设备时进入 100m 精度射校靶
    假如当前使用输入替身或 XR Device Simulator
    当测试输入确认“100m精度射校靶”
    那么系统应进入“100m任务说明”界面
    而且路由状态应与真实 XR 输入一致

  场景: 进入移动目标射击设置
    当玩家选择“移动目标射击”
    那么系统应进入“移动靶设置”界面
    而且当前训练模式应记录为移动目标射击

  场景: 进入堑壕射击地图选择
    当玩家选择“堑壕射击”
    那么系统应进入“堑壕地图选择”界面
    而且当前训练模式应记录为堑壕射击

  场景: 进入城镇攻防地图选择
    当玩家选择“城镇攻防”
    那么系统应进入“城镇地图选择”界面
    而且当前训练模式应记录为城镇攻防

  场景: 打开武器库
    当玩家选择“武器库”
    那么系统应进入“武器库 / 武器墙”界面
    而且当前已装备武器应高亮显示

  场景: 打开设置界面
    当玩家选择“设置”
    那么系统应进入“设置”界面
    而且设置界面应加载当前保存的配置

  场景: 快速重复点击菜单按钮
    假如玩家已经点击一个主菜单按钮
    当场景正在切换或页面正在打开
    那么系统应临时禁用主菜单按钮
    而且不应重复创建训练 Session 或重复加载场景
```

## Unity 验收点

- 主菜单按钮应绑定到 UI 路由，不直接创建训练逻辑。
- 可用 PlayMode Test 模拟 XR 射线点击每个按钮，并断言当前页面或状态机状态。
- P1 至少覆盖 100m 入口的 PlayMode Test；P2/P3 再覆盖后续模式入口的完整流程。
- 玩家档案数据建议由独立服务提供，避免写死在 UI 文本中。
