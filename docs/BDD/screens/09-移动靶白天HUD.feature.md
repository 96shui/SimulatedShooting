# 09 移动靶白天HUD

来源：draw.io 页面 `p09 09 移动靶白天HUD`

阶段：P2。

## 功能

作为玩家，我希望在白天移动靶 HUD 中使用觇孔完成 5 次点射，并看到弹药、命中、速度、方向和可射击状态。

```gherkin
# language: zh-CN
功能: 移动目标白天 HUD

  背景:
    假如玩家以白天模式开始移动目标射击
    而且目标速度为 4m/s
    那么系统应显示白天移动靶第一人称 HUD

  场景: 训练开始前等待 3 秒
    当 HUD 加载完成
    那么系统应显示等待开始状态
    而且目标在倒计时结束前不应移动
    而且玩家扳机输入不应计入射击记录

  场景: HUD 显示白天训练状态
    当目标开始从右向左移动
    那么 HUD 应显示弹药 10 发
    而且应显示点射模式
    而且应显示命中 0/10
    而且应显示目标速度 4m/s
    而且应显示方向右→左和状态可射击

  场景: 每次点射消耗两发弹
    假如目标处于可射击状态
    当玩家完成一次点射
    那么弹药应减少 2 发
    而且点射节奏条应记录一次点射
    而且最多允许 5 次点射

  场景: 命中目标后更新命中数
    当一次点射中的子弹命中侧身跑步靶
    那么命中数应增加对应命中发数
    而且命中反馈应在 HUD 中刷新

  场景: 目标到达左端后禁止射击
    假如目标到达左侧端点并进入 2 秒停留
    当玩家扣动扳机
    那么系统不应消耗弹药
    而且不应新增点射记录
    而且 HUD 应提示端点停留禁射

  场景: 目标抵达右侧终点后进入结算
    假如目标完成右→左→右运动
    当目标抵达右侧端点
    那么系统应结束训练
    而且进入“移动目标射击 训练结算”
```

## Unity 验收点

- 移动靶运动、端点停留和可射击状态应由状态机控制，测试可快进时间。
- 点射模式应一次消耗两发弹，不能变成全自动连发。
- P2 必须复用 P1 的输入适配，支持无 VR 输入替身和真实 XR 输入两条测试路径。
