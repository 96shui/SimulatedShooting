# 12 堑壕地图选择

来源：draw.io 页面 `p12 12 堑壕地图选择`

阶段：P3；P3 可先交付 1 张小型堑壕地图，第 2 张地图为后续扩展。

## 功能

作为玩家，我希望在进入堑壕射击前选择地图，并了解敌人数量、复杂度和任务条件。

```gherkin
# language: zh-CN
功能: 堑壕射击地图选择

  背景:
    假如玩家选择了堑壕射击
    那么系统应显示“堑壕射击 地图选择”

  场景: 默认选中堑壕地图 A
    当地图选择界面加载完成
    那么“堑壕地图 A”应处于选中状态
    而且界面应显示复杂度为中
    而且界面应显示敌人数量范围为 3-5

  场景大纲: 切换堑壕地图
    当玩家选择“<地图>”
    那么“<地图>”应显示选中边框
    而且另一个地图卡片应取消选中
    而且当前待确认地图应为“<地图>”

    例子:
      | 地图 |
      | 堑壕地图 A |
      | 堑壕地图 B |

  场景: 显示任务条件
    当玩家查看右侧任务条件
    那么界面应显示搜索完整堑壕
    而且显示消灭全部敌人
    而且显示失败条件为玩家死亡
    而且说明红色标记代表敌人预估区域

  场景: 选择地图进入武器库或任务准备
    假如玩家已选择堑壕地图 A
    当玩家点击“选择地图”
    那么系统应保存当前堑壕地图选择
    而且应进入“武器库 / 武器墙”或堑壕开场准备流程
    而且后续敌人生成数量应在 3 到 5 之间

  场景: 返回上一级
    当玩家点击“返回”
    那么系统应返回训练模式选择或主菜单
    而且不应创建堑壕任务 Session
```

## Unity 验收点

- 地图卡片只决定配置，不直接生成敌人；敌人应在任务 Session 创建时生成。
- 敌人实际位置应围绕红色预估区域随机偏移，并支持固定随机种子。
