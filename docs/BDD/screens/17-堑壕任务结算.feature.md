# 17 堑壕任务结算

来源：draw.io 页面 `p17 17 堑壕任务结算`

阶段：P3。

## 功能

作为玩家，我希望在堑壕任务结束后看到胜负、消灭敌人、搜索完成度、剩余弹药、队友状态、用时和堑壕平面图。

```gherkin
# language: zh-CN
功能: 堑壕射击任务结算

  背景:
    假如堑壕任务已经结束
    那么系统应显示“堑壕射击 任务结算”

  场景: 胜利条件达成
    假如玩家搜索完整堑壕
    而且消灭全部敌人
    而且玩家未死亡
    当系统结算任务
    那么任务结果应显示“胜利”
    而且消灭敌人数应显示为总敌人数/总敌人数
    而且搜索完成度应显示 100%

  场景: 玩家死亡导致失败
    假如玩家生命值归零
    当系统结算任务
    那么任务结果应显示“失败”
    而且结算界面仍应显示已消灭敌人、已搜索区域和用时

  场景: 显示堑壕任务统计
    当结算界面加载完成
    那么界面应显示地图名称
    而且显示消灭敌人数量
    而且显示搜索完成度
    而且显示剩余弹药
    而且显示队友状态
    而且显示用时

  场景: 显示堑壕平面图
    当结算界面加载完成
    那么右侧应显示堑壕平面图
    而且已搜索区域应以青蓝色标记
    而且敌人死亡位置或敌人标记应以红色叉号显示

  场景: 重新开始堑壕任务
    当玩家点击“重新开始”
    那么系统应清空当前堑壕任务 Session
    而且使用当前地图和武器重新进入堑壕开场或战斗准备

  场景: 返回主菜单
    当玩家点击“返回主菜单”
    那么系统应返回游戏主界面
    而且玩家档案应保存堑壕任务结果摘要
```

## Unity 验收点

- 结算数据应由 `TrenchSession` 汇总，包含敌人、搜索节点、弹药、队友和任务用时。
- 敌人尸体不消失是战斗场景行为，结算页可用平面图摘要表达。
- P3 结算只要求小型堑壕、简单敌人 AI、3 人纵队和简单血量闭环；高级队友指令结果可作为 Later 字段扩展。
