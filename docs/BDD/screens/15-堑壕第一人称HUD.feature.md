# 15 堑壕第一人称HUD

来源：draw.io 页面 `p15 15 堑壕第一人称HUD`

阶段：P3；换弹掩护、复杂拐角战术和自动拉回玩家属于 Later，除非单独升级阶段范围。

## 功能

作为玩家，我希望在堑壕第一人称战斗中查看小地图、弹药、队友状态、姿态和换肩信息，并完成搜索与战斗。

```gherkin
# language: zh-CN
功能: 堑壕第一人称 HUD

  背景:
    假如玩家已经进入堑壕任务
    而且玩家装备了容量 30 发的武器
    那么系统应显示堑壕第一人称 HUD

  场景: HUD 显示小地图和敌人预估点
    当堑壕 HUD 加载完成
    那么左上角小地图应显示玩家位置
    而且显示两名队友位置
    而且显示敌人预估点
    而且敌人预估点不应暴露精确随机偏移位置

  场景: HUD 显示弹药和弹匣信息
    当玩家持有满弹匣武器
    那么右侧弹药面板应显示“弹药 30 / 120”
    而且显示当前弹匣 30
    而且显示备用弹药 120

  场景大纲: 切换姿态
    当玩家选择姿态“<姿态>”
    那么玩家控制器应切换到“<姿态>”状态
    而且底部姿态栏应高亮“<姿态>”

    例子:
      | 姿态 |
      | 卧倒 |
      | 蹲下 |
      | 起立 |

  场景: 切换左右肩射击
    当玩家切换到左肩射击
    那么操作提示应显示换肩为左肩
    而且武器射击参考点应切换到左肩配置
    当玩家再次切换到右肩射击
    那么操作提示应显示换肩为右肩

  场景: 拐角处允许伸枪射击
    假如玩家位于可拐角射击区域
    当 HUD 显示“拐角射击：可用”
    而且玩家执行拐角伸枪动作并射击
    那么系统应允许武器从拐角外发射
    而且玩家身体不应完全暴露到拐角外

  场景: 换弹时二号队友上前掩护
    假如当前弹匣为空或玩家主动换弹
    当换弹动作开始
    那么二号队友应自动进入掩护状态
    而且队友状态面板应显示二号队友正在掩护或正常
    而且换弹完成后弹药面板应刷新

  场景: 玩家死亡后进入失败结算
    假如敌人攻击使玩家生命值归零
    当玩家死亡状态生效
    那么系统应结束堑壕任务
    而且进入“堑壕任务结算”
    而且任务结果应为失败
```

## Unity 验收点

- P3 敌人 AI 先实现出现、发现玩家、射击、死亡；高级巡逻、找掩体、投掷和包抄为 Later。
- P3 队友 AI 先实现 3 人纵队跟随；换弹掩护和复杂战术动作可在 Later 实现。
- 如已有 `Player.TakeDamage`、`EnemyAI` 或 `Firearm`，需接入任务状态、弹药状态和 HUD Presenter；没有时可先用可测试的原型组件实现。
- 小地图显示的是业务状态，不应仅是静态贴图。
