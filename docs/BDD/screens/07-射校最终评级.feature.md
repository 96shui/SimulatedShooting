# 07 射校最终评级

来源：draw.io 页面 `p07 07 射校最终评级`

阶段：P1。

## 功能

作为玩家，我希望在 100m 射校结束后看到最终评级、三轮记录和弹着缩略图，并能重新训练或返回模式选择。

```gherkin
# language: zh-CN
功能: 100m 射校最终评级

  背景:
    假如 100m 精度射校靶训练已经结束
    那么系统应显示“100m射校 训练结算”

  场景大纲: 根据通过轮次显示评级
    假如玩家在“<通过轮次>”满足 3 发全部进入 10 环
    当系统计算最终评级
    那么界面应显示评级“<评级>”

    例子:
      | 通过轮次 | 评级 |
      | 第1轮 | 优秀 |
      | 第2轮 | 良好 |
      | 第3轮 | 及格 |
      | 未通过 | 不及格 |

  场景: 显示三轮射击记录
    当结算界面加载完成
    那么界面应显示三轮射击记录
    而且已完成轮次应显示是否通过和弹着结果
    而且未使用轮次应显示“未使用”或等效状态

  场景: 评级来自服务层
    当结算界面加载完成
    那么最终评级应来自 100m 射校服务返回的结算 DTO
    而且 UI 不应根据文本或弹着缩略图自行计算评级

  场景: 显示弹着缩略图
    当结算界面加载完成
    那么弹着缩略图区域应显示靶图、10 环和命中点摘要
    而且缩略图应来自训练记录而不是固定示例图

  场景: 重新训练
    当玩家点击“重新训练”
    那么系统应清空当前 100m 射校 Session
    而且重新进入“100m任务说明”或直接重置到第 1 轮射击 HUD

  场景: 返回模式选择
    当玩家点击“返回模式选择”
    那么系统应返回“训练模式选择”
    而且玩家档案应保存本次评级摘要
```

## Unity 验收点

- 评级服务应可单元测试，输入为每轮是否通过和轮次，输出为评级。
- 结算保存玩家档案时要区分“最近评级”和完整训练记录。
- P1 PlayMode Test 必须覆盖无 VR 输入替身完成一轮、进入分析页，并在满足结束条件时进入结算页。
