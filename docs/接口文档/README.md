# VR射击训练游戏接口文档

本目录用于约束 UI 前端层与 Unity 玩法后端层之间的数据交互。这里的“前端”指 Unity UI、Canvas、Presenter、HUD 和输入适配层；“后端”指 Unity 内部玩法服务、Session、AI、武器、场景和持久化服务。

接口文档来源：

- `../BDD/术语表.md`
- `../../UI/Sample/vr-shooting-ui-reference-wireframes.drawio`
- `../需求/VR射击游戏需求文档_汇总版.md`
- `../需求/阶段化整体需求说明书.md`

## 分层约定

| 层 | 职责 | 不允许做的事 |
|---|---|---|
| UI View | 显示 TMP 文本、按钮、高亮、面板、HUD | 不计算评级、不生成敌人、不直接改训练规则 |
| UI Presenter | 把服务返回的 DTO 渲染到 View，转发 UI 事件 | 不持久化、不直接操作场景物体 |
| 应用服务 | 处理页面导航、训练 Session、设置、武器、结算 | 不引用具体 UI 控件 |
| 玩法服务 | 射击、弹着、移动靶、敌人、队友、搜索进度 | 不写 UI 文本 |
| Unity 场景层 | Prefab、Animator、XR Interactor、NavMesh、碰撞体 | 不持有业务规则常量的唯一来源 |

## 开发交付要求

- 后端服务必须先实现接口层和 DTO，再接入具体 UI。
- 所有 UI 显示数据必须来自 DTO 或事件，不允许在 View 中写死业务数值。
- 每个服务至少提供 EditMode 单元测试；涉及页面和场景跳转的接口提供 PlayMode 测试。
- BDD 场景是验收标准，接口文档是开发边界。如果两者冲突，先召开评审并更新 BDD。
