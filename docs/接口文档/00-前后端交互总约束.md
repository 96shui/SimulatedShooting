# 00 前后端交互总约束

## 目标

统一 UI 前端层与 Unity 后端玩法层之间的通信方式，保证 UI 可替换、玩法可测试、数据可追踪。

## 交互模式

UI 与后端采用“命令 + 查询 + 事件”模式：

- 命令：UI 请求后端执行行为，例如开始训练、应用设置、选择地图。
- 查询：UI 获取当前状态 DTO，例如当前 HUD、当前设置、当前武器。
- 事件：后端通知 UI 状态变化，例如页面切换、弹药变化、任务结算。

## 通用返回结构

```csharp
public readonly struct ServiceResult<T>
{
    public bool Success { get; init; }
    public T Data { get; init; }
    public ErrorCode ErrorCode { get; init; }
    public string Message { get; init; }
}
```

## 通用错误码

| 错误码 | 含义 | UI 处理 |
|---|---|---|
| None | 成功 | 正常渲染 |
| InvalidState | 当前状态不允许该操作 | 显示短提示，不切页 |
| InvalidInput | 入参非法 | 显示错误提示并记录日志 |
| NotFound | 配置、地图、武器或对象不存在 | 返回上一级或禁用入口 |
| Busy | 正在加载或执行中 | 保持按钮禁用 |
| Cooldown | 操作冷却中 | 显示冷却提示 |
| ResourceUnavailable | 资源未加载或依赖缺失 | 显示加载失败 |
| PersistenceFailed | 保存失败 | 提示玩家重试 |
| TestOnlyFailure | 测试替身故障 | 仅测试环境使用 |

## 通用事件

```csharp
public interface IGameEventBus
{
    void Publish<TEvent>(TEvent evt);
    IDisposable Subscribe<TEvent>(Action<TEvent> handler);
}
```

| 事件 | 触发方 | 订阅方 | 用途 |
|---|---|---|---|
| ScreenChangedEvent | UIRouter | UI 根节点、测试 | 页面切换 |
| SessionStartedEvent | TrainingSessionService | HUD、场景层 | 初始化训练 |
| SessionEndedEvent | 训练服务 | 结算页、档案服务 | 进入结算 |
| HudUpdatedEvent | HUD 服务 | HUD Presenter | 刷新 HUD |
| SettingsChangedEvent | 设置服务 | 音频、移动、HUD | 应用设置 |
| WeaponChangedEvent | 武器服务 | 武器生成器、UI | 装备武器 |
| SquadCommandIssuedEvent | 队友服务 | AI 层、HUD | 执行队友指令 |

## 版本和兼容

- DTO 增加字段必须设置默认值，避免旧 UI 崩溃。
- 枚举新增项必须在 UI 端有默认显示策略。
- 删除字段或改名需要同步更新 BDD、接口文档和测试。

## 幂等要求

以下命令必须幂等：

- 应用同一轮射校调整
- 保存相同设置
- 关闭当前页面
- 重复点击已经禁用的开始按钮
- 重复请求当前 HUD DTO

## 随机性约束

所有随机行为必须支持固定种子：

- 100m 射校初始弹着偏移
- 堑壕敌人数量和位置偏移
- 城镇敌人数量和房间分布
- AI 遭遇触发点

```csharp
public readonly struct RandomSeed
{
    public int Value { get; init; }
    public bool IsFixed { get; init; }
}
```

## 前后端责任边界

- UI 可以请求“开始训练”，但不能直接创建敌人、弹药、队友。
- UI 可以显示“可射击/禁止射击”，但不能决定射击是否有效。
- UI 可以显示“胜利/失败”，但不能计算胜负。
- UI 可以临时修改设置，但只有设置服务可以持久化。
