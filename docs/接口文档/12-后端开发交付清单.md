# 12 后端开发交付清单

## 目标

把接口文档转成 Unity 后端开发团队可执行的交付顺序，确保开发完成后可以被 BDD 验收。

## 第一阶段：基础框架

| 任务 | 产出 | 验收 |
|---|---|---|
| 建立通用 DTO 和错误码 | `ServiceResult<T>`、`ErrorCode`、通用枚举 | EditMode Test 覆盖成功/失败返回 |
| 建立事件总线 | `IGameEventBus` | 可订阅、发布、取消订阅 |
| 建立路由服务 | `IUIRouter` | 可从主菜单跳到各页面 |
| 建立 Session 服务 | `ITrainingSessionService` | 可创建、开始、暂停、结束 Session |
| 建立测试 ID 组件 | `UITestId` | PlayMode Test 可按 ID 找到 UI |

## 第二阶段：全局服务

| 任务 | 产出 | 验收 |
|---|---|---|
| 设置服务 | `ISettingsService` | 修改、应用、恢复默认、返回不保存 |
| 武器服务 | `IWeaponService` | 选择预览、装备、属性显示 |
| 弹药服务 | `IAmmoService` | 消耗、换弹、HUD 刷新 |
| HUD 聚合服务 | `IHUDService` | 根据 Session 返回 HUD DTO |

## 第三阶段：训练玩法服务

| 模块 | 必须实现 |
|---|---|
| 100m 射校 | 固定偏移、3 发一轮、弹着记录、偏差分析、应用调整、评级 |
| 移动靶 | 昼夜速度、3 秒等待、路线运动、端点禁射、5 次点射、评级 |
| 堑壕 | 地图配置、3-5 敌人、搜索节点、队友队形、胜负结算 |
| 城镇 | 街道敌人、建筑房间、楼层小地图、房间搜索、胜负结算 |
| 队友指令 | 环形菜单命令、手榴弹、火力支援、掩护换弹、前进、停止 |

## 第四阶段：UI 和场景接入

| 接入点 | 要求 |
|---|---|
| 页面根节点 | 命名为 `Screen_<ScreenId>` |
| 按钮 | 命名为 `Button_<ScreenId>_<Action>`，绑定到 UIEvent |
| HUD 文本 | 命名为 `Text_<ScreenId>_<Data>` 或 `Hud_<HudType>_<Name>` |
| 场景切换 | 通过 `UIRouter` 和 `TrainingSessionService` 协调 |
| XR 输入 | 通过 `IXRTrainingInput` 适配，不在业务服务里读取硬件输入 |

## 第五阶段：测试与验收

| 测试类型 | 覆盖 |
|---|---|
| EditMode Test | DTO、算法、评级、设置、武器、Session 状态机 |
| PlayMode Test | 页面跳转、按钮点击、HUD 刷新、场景加载、训练闭环 |
| VR 实机验收 | 可读性、舒适度、射击手感、队友遮挡、HUD 视野 |

## 功能级交付门禁

每个拆分功能完成时，必须同时满足以下条件：

1. 能追溯到至少一个 `docs/BDD/screens/*.feature.md` 场景。
2. 已实现或更新对应接口文档中的 DTO、服务方法、事件和错误码。
3. 纯业务规则、算法、DTO、状态机和服务错误码有 EditMode 单元测试。
4. UI、场景、按钮、HUD、XR 输入替身或训练闭环有 PlayMode 测试。
5. 测试能在没有 VR 设备时运行；真实 VR 设备相关体验另列手工验收项。
6. 测试缺失时，该功能不得进入“完成”状态，只能标记为未完成、部分完成或待补测。

## 产品阶段映射

技术交付顺序可以按基础框架、全局服务、玩法服务、UI/场景接入推进；产品范围按以下三阶段验收：

| 产品阶段 | 必须交付 | 暂不要求 |
|---|---|---|
| P1：100m 射校原型闭环 | 基础 DTO/错误码、事件总线、路由、Session、HUD 服务、输入适配、基础武器、按钮换弹、左右肩切换、100m 射校服务、100m UI/HUD/弹着分析/结算、对应 EditMode/PlayMode 测试 | 移动靶、武器库完整功能、设置完整功能、敌人 AI、队友 AI、堑壕、城镇、真实弹匣插拔 |
| P2：移动靶与全局系统扩展 | 移动靶服务、昼夜/速度选择、端点禁射、点射、移动靶结算、武器库原型、设置原型、弹药/HUD 复用、对应测试 | 堑壕/城镇完整战斗、高级 AI、高级队友指令 |
| P3：小型战斗原型 | 1 张小型堑壕、1 个小型城镇、简单敌人 AI、3 人纵队、简单血量、小地图、搜索进度、胜负结算、对应测试 | 第 2 张地图、高级敌人 AI、高级队友 AI、手榴弹/火力支援/拐角拉回、复杂医疗 |
 
任何实现任务必须先标注所属产品阶段。Later 功能不得作为 P1/P2/P3 交付的隐性前置条件。

## 最小可交付版本

后端团队第一轮 P1 最小可交付应包含：

1. 主菜单到 100m 任务说明的路由。
2. 无 VR 输入替身和真实 XR 输入共用的输入适配接口。
3. 基础武器：双手持枪、扳机开火、按钮换弹、左右肩切换、简单后坐力。
4. 100m 射校完整闭环：任务说明、HUD、3 发一轮、弹着分析、应用调整、最终评级。
5. 100m HUD 数据和结算数据都来自服务层 DTO 或事件。
6. 对应 EditMode/PlayMode 测试，且无 VR 设备时可运行。

## 禁止事项

- 不允许 UI 直接计算评级。
- 不允许按钮直接创建敌人或修改场景对象。
- 不允许 HUD 显示写死示例值。
- 不允许随机逻辑没有固定种子入口。
- 不允许训练结算从 UI 文本反推数据。
