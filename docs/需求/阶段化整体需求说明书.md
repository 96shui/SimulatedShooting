# VR射击训练游戏阶段化整体需求说明书

## 总体定位

本项目是一款 Unity VR 射击训练游戏原型，面向现代军事训练模拟器方向。第一目标是在 Unity Editor 中完成可演示、可测试的训练闭环；VR 设备到位后，同一套功能需要支持真实头显和手柄验证。

画风尽量写实，优先保证场景比例、武器观感、射击弹道和后坐力可信。模型和贴图精度需要控制，避免影响 VR 帧率和迭代速度。

## 测试与输入要求

- 无 VR 设备时，必须能通过 Unity Editor、XR Device Simulator 或输入测试替身完成核心流程测试。
- 有 VR 设备时，必须能通过真实头显和手柄测试双手持枪、扳机开火、按钮换弹、左右肩切换和 HUD 可读性。
- 所有输入应通过可替换的输入适配层进入业务服务，业务服务不直接读取硬件输入。

## BDD 协作要求

项目高度使用 AI 协作，所有任务必须遵循当前 BDD 工作流：

1. 先明确需求范围。
2. 更新或确认 `docs/BDD/screens/*.feature.md`。
3. 更新或确认 `docs/接口文档/*.md`。
4. 再实现代码、场景、UI 和测试。
5. 行为或接口变化时，同步更新 BDD 和接口文档。

## 第一阶段：100m 射校原型闭环

目标：完成一个可演示、可测试、可扩展的 100m 精度射校训练闭环。

### 必须实现

- 主菜单进入 100m 精度射校靶。
- 100m 任务说明页面。
- 第一人称射击 HUD。
- 双手持枪。
- 扳机开火。
- 按按钮换弹。
- 左右肩切换。
- 简单但可感知的后坐力和枪口抖动。
- 弹着点由玩家瞄准稳定性决定，不做纯随机散布。
- 开局生成一次固定弹着偏移，三轮共用。
- 每轮 3 发，共最多 3 轮。
- 每轮结束显示弹着分析：上下/左右偏差、准星柱调整方向和角度、觇孔调整方向和格数。
- 玩家应用调整后进入下一轮。
- 调整后 3 发全部进入 10 环则通过。
- 评级规则：第 1 轮通过为优秀，第 2 轮为良好，第 3 轮为及格，3 轮未通过为不及格。

### 基础服务

- `ServiceResult<T>`
- `ErrorCode`
- `IGameEventBus`
- `IUIRouter`
- `ITrainingSessionService`
- `IHUDService`
- `IZeroingService`
- 输入适配接口
- 基础武器控制和弹药消耗

### 场景范围

- 小型 100m 靶场。
- 写实风格原型资产。
- 包含射击位、靶标、基础环境、光照和测试出生点。

### 暂不实现

- 移动靶。
- 敌人 AI。
- 队友 AI。
- 堑壕和城镇战斗。
- 历史训练记录。
- 真实弹匣插拔、拉机柄、保险、卡弹等高拟真枪械操作。

### 验收标准

- 无 VR 设备时可在 Unity Editor 完成完整 100m 射校流程。
- 有 VR 设备时可完成同样流程。
- HUD 数据来自服务层，不写死示例值。
- 评级和调整计算由服务层完成。
- EditMode 测试覆盖偏差计算、调整计算、评级规则。
- PlayMode 测试覆盖进入训练、完成一轮、进入分析页、进入结算页。

## 第二阶段：移动靶与全局系统扩展

目标：在 100m 射校基础上扩展第二个训练模式，并补齐可复用系统。

### 必须实现

- 移动目标射击模式。
- 昼夜模式选择。
- 速度选择：白天 3m/s、4m/s、5m/s；夜晚 2m/s、2.5m/s、3m/s。
- 进入射击视角后等待 3 秒，靶子开始运动。
- 靶子从右向左运动，到左端停留 2 秒，停留期间禁止射击，再返回右端结束。
- 点射模式：一次出两发，共 10 发，5 次点射。
- 白天使用觇孔。
- 夜晚使用微光瞄准镜。
- 按命中数评级：3 发及格，4 发良好，5 发及以上优秀。

### 全局系统

- 武器库原型，至少支持训练用主武器。
- 设置界面原型：移动方式、转向方式、HUD 透明度、音量、简单保存。
- HUD 聚合服务支持 100m 和移动靶。
- 弹药服务支持当前弹匣、备用弹药、换弹、禁射不消耗弹药。

### 场景范围

- 小型移动靶靶场。
- 左右端点标识。
- 白天/夜晚通过灯光、材质或后处理区分。
- 夜视镜先实现可用效果，后续再提升真实感。

### 验收标准

- 100m 射校仍可稳定运行。
- 移动靶可在无 VR 和有 VR 两种方式下测试。
- 靶子运动、端点禁射、点射弹药消耗、命中评级由服务控制。
- EditMode 测试覆盖移动靶状态机、命中评级、端点禁射。
- PlayMode 测试覆盖模式选择、开始训练、HUD 刷新、结算跳转。

## 第三阶段：小型堑壕与城镇战斗原型

目标：完成小规模战术训练玩法，先验证敌人、队友、搜索、胜负闭环。

### 堑壕射击

- 先做 1 张小型堑壕地图，后续扩展到 2 张。
- 玩家和两名队友组成 3 人纵队。
- 玩家在最前，第二名队友在玩家后方朝前，第三名队友在最后朝后警戒。
- 敌人简单 AI：开局生成 3-5 个，出现在预设点附近，发现玩家后射击，被命中后死亡，尸体不消失。
- 小地图显示玩家、队友和敌人标红位置。
- 搜索完堑壕并消灭所有敌人则胜利。
- 玩家血量归零则失败。

### 城镇攻防

- 先做小型城镇场景：一条街、一栋 3 层楼、3-5 个可搜索房间。
- 街道敌人 1-2 名。
- 建筑敌人 3-6 名。
- 敌人分布在楼道拐角和房间内。
- 搜索完楼房并消灭所有敌人则胜利。
- 玩家血量归零则失败。

### 战术动作

- 姿态先做站立、蹲下、卧倒状态切换和视角高度变化。
- 换肩射击沿用前两阶段武器系统。
- 拐角射击先做简化版，允许玩家在特定拐角位置伸枪射击。
- 队友指令延后，后续再加手榴弹、火力支援、掩护换弹。

### 暂不实现

- 高级敌人 AI：巡逻、找掩体、包抄、复杂投掷逻辑。
- 高级队友 AI：自动掩护、拉回玩家、火力压制、投掷手榴弹。
- 大型开放地图。
- 复杂伤害部位系统。
- 医疗救援系统。

### 验收标准

- 堑壕小地图、搜索进度、敌人清除、胜负结算可跑通。
- 城镇街道、进入建筑、房间搜索、敌人清除、胜负结算可跑通。
- 简单血量机制有效。
- 敌人和队友逻辑由服务或 AI 层控制，不由 UI 按钮直接创建或修改。
- 无 VR 设备时仍可通过模拟输入完成测试。
- 有 VR 设备时验证移动、瞄准、HUD 可读性和基本射击手感。

## 四人分工

### 场景负责人

- 100m 靶场、移动靶靶场、小型堑壕、小型城镇。
- 灯光、材质、NavMesh、出生点、掩体、性能优化。
- 控制模型和贴图精度，保证 VR 原型流畅。

### UI 负责人

- 主菜单、模式选择、100m 任务说明、HUD、弹着分析、评级。
- 移动靶设置、HUD、结算。
- 武器库、设置界面。
- 堑壕/城镇 HUD、小地图、结算。
- 所有 UI 按 BDD 命名和测试 ID 规范实现。

### 功能负责人 A

- 通用 DTO、错误码、事件总线、页面路由。
- Session 服务、HUD 服务。
- 100m 射校服务、移动靶服务。
- 对应 EditMode 和 PlayMode 测试。

### 功能负责人 B

- 输入适配、武器控制、双手持枪。
- 扳机开火、按钮换弹、左右肩切换。
- 后坐力和弹道。
- 敌人简单 AI、队友 3 人纵队。
- 血量、伤害、命中反馈。
