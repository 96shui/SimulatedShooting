# 13 武器库武器墙

来源：draw.io 页面 `p13 13 武器库武器墙`

## 功能

作为玩家，我希望在武器库中查看并装备不同武器，使后续堑壕或城镇训练使用选中的武器。

```gherkin
# language: zh-CN
功能: 武器库 / 武器墙

  背景:
    假如玩家打开武器库
    那么系统应显示四把可选武器

  场景: 默认显示当前已装备武器
    假如玩家当前已装备 95-1 自动步枪
    当武器库界面加载完成
    那么“95-1自动步枪”应显示为选中
    而且右侧武器属性应显示名称、类型、弹匣容量、后坐力和适用场景

  场景大纲: 选择武器后刷新属性
    当玩家选择“<武器>”
    那么“<武器>”应显示为选中
    而且右侧属性面板应显示该武器的属性
    而且“装备”按钮应可用

    例子:
      | 武器 |
      | 19-1自动步枪 |
      | 95-1自动步枪 |
      | QBS-09霰弹枪 |
      | QJB-201班用机枪 |

  场景: 装备选中的武器
    假如玩家已选择“95-1自动步枪”
    当玩家点击“装备”
    那么系统应将当前武器设置为 95-1 自动步枪
    而且后续进入堑壕或城镇训练时应生成该武器
    而且初始弹药应符合该武器弹匣容量和备用弹药规则

  场景: 返回上一界面
    当玩家点击“返回”
    那么系统应返回打开武器库前的页面
    而且如果玩家未点击“装备”，当前已装备武器不应改变

  场景: 不适用场景的武器提示
    假如某武器不适用于当前训练模式
    当玩家选择该武器
    那么界面应提示适用场景限制
    而且“装备”按钮应禁用或要求玩家确认
```

## Unity 验收点

- 武器列表和属性建议来自 `WeaponDefinition` 配置，不应硬编码在 UI。
- `装备` 行为应写入 `WeaponLoadoutService`，由训练场景读取。
