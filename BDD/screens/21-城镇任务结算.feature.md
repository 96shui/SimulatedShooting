# 21 城镇任务结算

来源：draw.io 页面 `p21 21 城镇任务结算`

## 功能

作为玩家，我希望在城镇攻防结束后看到胜负、街道清除、建筑搜索完成度、房间搜索、消灭敌人、剩余弹药和队友状态。

```gherkin
# language: zh-CN
功能: 城镇攻防任务结算

  背景:
    假如城镇攻防任务已经结束
    那么系统应显示“城镇攻防 任务结算”

  场景: 胜利条件达成
    假如街道敌人已清除
    而且建筑内所有房间已搜索
    而且所有敌人已消灭
    而且玩家未死亡
    当系统结算任务
    那么任务结果应显示“胜利”
    而且建筑搜索完成度应显示 100%

  场景: 玩家死亡导致失败
    假如玩家生命值归零
    当系统结算任务
    那么任务结果应显示“失败”
    而且结算界面仍应显示已完成的街道清除、房间搜索和消灭敌人数量

  场景: 显示城镇任务统计
    当结算界面加载完成
    那么界面应显示街道敌人清除状态
    而且显示建筑搜索完成度
    而且显示房间搜索数量
    而且显示消灭敌人数量
    而且显示剩余弹药
    而且显示队友状态

  场景: 显示三层楼平面缩略图
    当结算界面加载完成
    那么右侧应显示 1F、2F、3F 平面缩略图
    而且已搜索房间应以青蓝色标记
    而且敌人位置应以红色叉号标记

  场景: 重新开始城镇任务
    当玩家点击“重新开始”
    那么系统应清空当前城镇任务 Session
    而且使用当前地图和武器重新进入城镇任务准备或街道阶段

  场景: 返回主菜单
    当玩家点击“返回主菜单”
    那么系统应返回游戏主界面
    而且玩家档案应保存城镇任务结果摘要
```

## Unity 验收点

- 结算页应汇总 `UrbanSession`，不要从 UI 文本反向计算结果。
- 房间搜索数和敌人消灭数要能处理失败中途结算。
