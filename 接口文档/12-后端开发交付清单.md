# 12 后端开发交付清单

## 目标

把接口文档转成 Unity 后端开发团队可执行的交付顺序，确保开发完成后可以被 BDD 验收。

## 第一阶段：基础框架

| 任务 | 产出 | 验收 |
|---|---|---|
| 建立通用 DTO 和错误码 | `ServiceResult<T>`、`ErrorCode`、通用枚举 | EditMode Test 覆盖成功/失败返回 |
| 建立事件总线 | `IGameEventBus` | 可订阅、发布、取消订阅 |
| 建立路由服务 | `IUIRouter` | 可从主菜单跳到各页面 |
| 建立 Session 服务 | `ITrainingSessionService` | 可创建、开始、暂停、结束 Session |
| 建立测试 ID 组件 | `UITestId` | PlayMode Test 可按 ID 找到 UI |

## 第二阶段：全局服务

| 任务 | 产出 | 验收 |
|---|---|---|
| 设置服务 | `ISettingsService` | 修改、应用、恢复默认、返回不保存 |
| 武器服务 | `IWeaponService` | 选择预览、装备、属性显示 |
| 弹药服务 | `IAmmoService` | 消耗、换弹、HUD 刷新 |
| HUD 聚合服务 | `IHUDService` | 根据 Session 返回 HUD DTO |

## 第三阶段：训练玩法服务

| 模块 | 必须实现 |
|---|---|
| 100m 射校 | 固定偏移、3 发一轮、弹着记录、偏差分析、应用调整、评级 |
| 移动靶 | 昼夜速度、3 秒等待、路线运动、端点禁射、5 次点射、评级 |
| 堑壕 | 地图配置、3-5 敌人、搜索节点、队友队形、胜负结算 |
| 城镇 | 街道敌人、建筑房间、楼层小地图、房间搜索、胜负结算 |
| 队友指令 | 环形菜单命令、手榴弹、火力支援、掩护换弹、前进、停止 |

## 第四阶段：UI 和场景接入

| 接入点 | 要求 |
|---|---|
| 页面根节点 | 命名为 `Screen_<ScreenId>` |
| 按钮 | 命名为 `Button_<ScreenId>_<Action>`，绑定到 UIEvent |
| HUD 文本 | 命名为 `Text_<ScreenId>_<Data>` 或 `Hud_<HudType>_<Name>` |
| 场景切换 | 通过 `UIRouter` 和 `TrainingSessionService` 协调 |
| XR 输入 | 通过 `IXRTrainingInput` 适配，不在业务服务里读取硬件输入 |

## 第五阶段：测试与验收

| 测试类型 | 覆盖 |
|---|---|
| EditMode Test | DTO、算法、评级、设置、武器、Session 状态机 |
| PlayMode Test | 页面跳转、按钮点击、HUD 刷新、场景加载、训练闭环 |
| VR 实机验收 | 可读性、舒适度、射击手感、队友遮挡、HUD 视野 |

## 最小可交付版本

后端团队第一轮最小可交付应包含：

1. 主菜单到所有页面的路由。
2. 武器库可装备武器。
3. 设置可应用和持久化。
4. 100m 射校完整闭环。
5. 移动靶完整闭环。
6. 堑壕和城镇至少完成地图选择、HUD 数据和结算 DTO。

## 禁止事项

- 不允许 UI 直接计算评级。
- 不允许按钮直接创建敌人或修改场景对象。
- 不允许 HUD 显示写死示例值。
- 不允许随机逻辑没有固定种子入口。
- 不允许训练结算从 UI 文本反推数据。
